# Roadmap  
  
这个项目的目标，不是一次性做成终局，  
而是围绕**一人制公司与 Agent 智能能力结合**这条主线，  
逐步形成一个更清楚、更可运行、更可沉淀的开源创新共同体。  
  
---  
  
## Phase 1：立住骨架  
  
目标：  
先把项目最小骨架立起来，让后来者能够进入、理解并开始使用。  
  
本阶段重点包括：  
  
- 建立 README 与 INDEX  
- 明确愿景、使命与社区总纲  
- 明确一人制公司关键特征  
- 建立主经营智能体定义  
- 建立七核岗位系统总览  
- 建立最小工作流与协议模板  
- 开启 Discussions  
- 发布第一版 use cases  
  
状态：  
**进行中**  
  
---  
  
## Phase 2：补齐骨架与样板  
  
目标：  
让项目从“有门面”升级为“有完整方法骨架和真实样板”。  
  
本阶段重点包括：  
  
- 补齐七张角色卡  
- 补齐七核协作工作流  
- 补齐审计、留痕、复盘协议  
- 补齐内容 / 研究 / 项目推进三类样板  
- 补齐社区导读、共建路径、讨论结构  
- 补齐第一批模板系统  
  
状态：  
**下一阶段重点**  
  
---  
  
## Phase 3：形成最小可运行系统  
  
目标：  
让用户不仅能阅读，还能真正开始使用这套结构推进真实任务。  
  
本阶段重点包括：  
  
- 完善任务进入模板  
- 完善任务结构化模板  
- 完善任务模式（轻任务 / 标准任务 / 高风险任务）  
- 完善交接清单  
- 优化 INDEX 导航  
- 增加更多真实任务样板  
- 收集真实反馈并修正骨架  
  
状态：  
**待启动**  
  
---  
  
## Phase 4：沉淀经营反馈机制  
  
目标：  
让系统不仅“能跑”，还开始具备“经营判断”。  
  
本阶段重点包括：  
  
- 时间盈亏反馈  
- 质量盈亏反馈  
- 资源盈亏反馈  
- 商业相关度反馈  
- 周期性经营摘要模板  
- 主经营智能体的反馈机制增强  
  
状态：  
**待启动**  
  
---  
  
## Phase 5：进入工具化与工作台探索  
  
目标：  
在方法骨架足够稳定后，探索更轻量、更实用的工具与工作台形态。  
  
本阶段重点包括：  
  
- 结构图与可视化  
- 模板工具化  
- 工作台原型  
- Discussions 与模板系统联动  
- 更适合工程实现者进入的接口设计  
  
状态：  
**远期方向**  
  
---  
  
## 长期方向  
  
长期来看，这个项目会逐步长向：  
  
- 一人制公司智能经营系统  
- 主经营智能体的真实接口化  
- 七核岗位系统的更强协议化与工具化  
- 场景化经营样板库  
- 更成熟的开源社区与协作网络  
  
---  
  
## 当前最优先事项  
  
当前阶段，最优先的是：  
  
1. 立住首页门面  
2. 开启 Discussions  
3. 上传第一批核心文件  
4. 补齐角色卡与样板  
5. 让项目从“好看”进入“可运行”  
  
---  
  
## 一句话总结  
  
**先把骨架立住，再把样板跑通，再让系统逐步形成经营能力。**

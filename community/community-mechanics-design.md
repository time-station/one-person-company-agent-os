## 社区四种核心角色

### 🔬 实验者（Experimenter）
用自己的真实业务跑这套系统，记录结果，贡献原始数据。
**最低要求：** 完整跑过一次七核系统，提交一份复盘报告。
**贡献物：** 使用报告、问题报告、模板改进建议。

### 🏗️ 构建者（Builder）
用代码或文档把想法变成现实。
**最低要求：** 提交一个被合并的功能/文档PR。
**贡献物：** 代码实现、新协议、工具集成、演示系统。

### 🧠 理论家（Theorist）
从更深的理论视角思考系统的完整性和边界。
**最低要求：** 发布一篇有实质内容的分析文章，被社区引用。
**贡献物：** 概念分析、框架批判、架构建议。

### 🌉 布道者（Evangelist）
把这套系统带给更多类型的一人公司主理人。
**最低要求：** 引导至少一个真实主理人完整使用系统。
**贡献物：** 案例研究、入门教程、特定行业适配。

## 社区内容质量标准

**黄金标准：一份提交应该包含……**

```
1. 任务类型（研究/决策/创作/执行）
2. 使用的系统版本和配置
3. 实际输入（主理人的原始问题）
4. 系统输出（各核的实际输出）
5. 你的判断：哪些有效，哪些没效，为什么
6. 修改建议（如果有）
7. 可复用资产（模板/框架/协议，如果产出了）
```

**被直接关闭的提交类型：**
- 纯粹的"这个系统很棒"（无内容）
- 未实际使用就给出的架构建议
- 工具推荐（不是这个社区的焦点）
- 与一人制公司运营无关的通用AI讨论

## 核心产出物：社区共识协议库

社区最有价值的集体产出不是讨论，而是**协议**。

每个经过验证的协议包含：
```yaml
协议名称: 竞品快速研究协议 v2.1
适用场景: 新市场进入判断
使用次数: 47
成功率: 82%  # 主理人反馈"达成决策"的比例
平均完成时间: 23分钟
核心框架:
  - 功能矩阵（5维）
  - 用户评价信号（来源权重）
  - 价格定位三角
已知局限:
  - 对新兴市场预测准确率下降
  - B2B场景需要增补"销售流程"维度
贡献者: 社区成员（匿名）
最后更新: 2026-03
```

**目标：** 一年内形成 30 个经过社区验证的核心协议。

## 匿名共鸣网络机制

**隐私第一，集体学习第二。**

### 数据处理原则
- 所有业务内容（客户名称、产品细节、财务数据）绝不共享
- 只共享结构化的"框架使用情况"和"模式标签"
- 主理人主动选择参与，可随时退出并删除历史数据
- 开源代码，任何人可审计数据处理逻辑

### 可以聚合的维度
```
任务类型分布 → 了解"一人公司最常做什么类型的研究"
框架使用率 → 了解"哪些分析框架真正被复用"
判断核分级分布 → 了解"什么比例的任务被判断为值得做"
红队核发现 → 去识别化后的"最常见假设盲点类型"
系统完整运行率 → 了解"七核有多少任务跑完了全部环节"
```

### 主理人可以查看
- "与我画像相似的主理人，最常使用哪些框架"
- "我的任务类型分布与社区平均的差异"
- "我的系统使用完整率在社区中的分位"

## 社区节律

**月度：社区内容日历**

| 周 | 焦点 |
|---|---|
| 第一周 | 新实验报告收集（上月使用报告） |
| 第二周 | 框架迭代讨论（基于上月报告的改进提案） |
| 第三周 | 构建者进展分享（代码/文档PR展示） |
| 第四周 | 月度协议更新（投票确定进入协议库的内容） |

**季度：系统回顾**
- 重新审视核心框架是否仍然成立
- 引入被质疑最多的假设，公开讨论
- 发布《季度系统健康报告》

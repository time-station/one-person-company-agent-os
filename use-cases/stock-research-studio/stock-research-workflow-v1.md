# Stock Research Workflow v1

## 股票研究工作流

这份文档描述的是：
**一个完整的股票研究任务，如何在七核系统中高效运转。**

它不是理论描述，而是可以直接跟着跑的操作流程。

---

## 一、工作流总图

```
主理人提出研究意图
        ↓
主经营智能体接收并分级
        ↓
判断 Agent：值得研究吗？
   ↓ 是          ↓ 否
编排 Agent：     → 止损，记录原因
拆出研究任务书
        ↓
研究 Agent：
搜集财务/行业/估值数据
        ↓
表达 Agent：
形成结构化研究报告
        ↓
监督审计 Agent：
逻辑偏差审查
   ↓ 通过         ↓ 发现问题
执行 Agent：     → 返回修正
决策落地与跟踪
        ↓
记忆 Agent：
沉淀公司档案与研究经验
        ↓
主理人复核 + 反馈
```

---

## 二、完整流程详解

### 阶段 0：研究意图进入（主理人 → 主经营智能体）

**主理人提供以下任一形式的输入：**

```
形式一：直接指定标的
「研究一下 NVIDIA，我想知道当前的估值是否合理」

形式二：条件筛选
「帮我找几家过去3年 ROE 持续 > 20% 的消费品公司，做初步筛查」

形式三：行业入口
「我想了解国内新能源储能行业，从哪几家公司开始研究比较好？」

形式四：更新旧研究
「我之前研究过腾讯，但上次是1年前，现在需要更新一下判断」
```

**主经营智能体做的事：**
- 确认研究目的（长期持有 / 交易机会 / 行业地图）
- 初步分级（轻看 / 标准 / 深度）
- 设定研究截止时间（防止无边界研究）
- 调度判断 Agent 进入

---

### 阶段 1：判断前置（判断 Agent）

**输入**：研究标的 + 研究目的

**判断 Agent 执行以下检查：**

```
检查项 1：能力圈匹配
□ 主理人是否了解该行业的基本运作逻辑？
□ 如果不了解，是否需要先做行业入门研究？

检查项 2：信息可获取性
□ 是否有充足的公开信息（财报、研报、新闻）？
□ 是否存在信息黑箱（如：VIE结构复杂、审计不透明）？

检查项 3：研究时机
□ 是否在财报季（信息最新）？
□ 是否刚发生重大事件（格局变化期）？
□ 估值是否处于明显极端区间（历史高/低位）？

检查项 4：明显拦截信号
□ 是否有持续的监管诉讼风险？
□ 是否有财务欺诈历史？
□ 流动性是否过低？
```

**判断输出：**

| 结论 | 含义 | 下一步 |
|------|------|--------|
| 深入研究 | 值得做标准或深度研究 | 进入编排 Agent |
| 初步筛查 | 先快速看一遍，决定是否深入 | 进入快速筛查流程 |
| 列入观察清单 | 现在不是最佳研究时机 | 记录，定期复查 |
| 跳过 | 不在能力圈 / 信息不可靠 / 明显拦截信号 | 止损，记录原因 |

---

### 阶段 2：研究结构化（编排 Agent）

**输入**：判断 Agent 的通过结论 + 研究深度

**编排 Agent 产出研究任务书，包含：**

```
1. 研究框架选择
   价值框架：商业模式质量 + 护城河 + 长期FCF
   成长框架：TAM + 增速 + 市占率提升空间
   困境反转框架：催化剂 + 估值底部 + 修复路径
   周期框架：供需格局 + 库存周期 + 价格弹性

2. 核心问题（3-5个）
   每个问题明确：
   - 问题是什么
   - 需要什么类型的证据
   - 答案会如何影响最终结论

3. 研究边界
   本轮覆盖：[明确说明]
   本轮不做：[明确说明]

4. 证据需求清单
   □ 财务数据（具体到哪些指标）
   □ 行业数据（具体到什么维度）
   □ 管理层信息（具体到哪些来源）
   □ 估值数据（具体到哪种方法）

5. 主理人仲裁点
   哪些判断必须由主理人拍板（不能由 Agent 代劳）
```

---

### 阶段 3：数据搜集（研究 Agent）

**输入**：研究任务书

**研究 Agent 按任务书搜集以下数据（根据深度调整范围）：**

#### 初步筛查数据包（1-2小时）
```
□ 公司主业一句话描述
□ 近3年营收/净利润增速
□ 当前PE/PB vs. 历史均值
□ 是否有重大负面新闻
□ 行业地位（市场第几名）
```

#### 标准研究数据包（3-6小时）
```
财务层（5年数据）：
□ 营收/净利润增速趋势
□ 毛利率/净利率趋势
□ FCF计算与质量分析
□ ROIC趋势
□ 资产负债健康度评估

行业层：
□ 市占率变化（近3年）
□ 核心竞争对手对比表（3家）
□ 行业增速 vs. 公司增速

管理层层：
□ 最近2次财报电话会议摘要
□ 大股东持股变化（近6个月）
□ 资本配置历史评估

估值层：
□ 历史PE/EV-EBITDA区间
□ 当前估值百分位
□ 卖方共识预期
```

#### 深度研究数据包（1-3天）
```
在标准研究基础上增加：
□ 10年以上财务历史穿透
□ 完整DCF估值模型
□ 全部竞争对手深度对比
□ 管理层历史发言系统梳理
□ 供应链上下游访谈/调研
□ 多情景假设与压力测试
□ 全面风险矩阵建立
```

---

### 阶段 4：报告形成（表达 Agent）

**输入**：原始数据包 + 研究任务书

**表达 Agent 遵循以下原则：**

**原则一：结论前置**
报告的第一段就是结论，而不是把结论藏在最后。

**原则二：证据驱动**
每个判断都有对应的证据，没有证据的判断标注为"假设"。

**原则三：不确定性诚实**
不装确定，不假精确。用"大概率"/"可能"/"存在不确定性"诚实表述。

**原则四：退出条件必须写明**
每个买入/持有结论，都必须包含"逻辑失效条件"。

**标准报告时间要求：**
- 初步筛查报告：500字以内
- 标准研究报告：1500-3000字
- 深度研究报告：3000-8000字

---

### 阶段 5：审计（监督审计 Agent）

**输入**：完整研究报告

**审计 Agent 执行标准审计清单：**

```
逻辑审计：
□ 结论与证据是否一致？
□ 有无"先有结论再找证据"的反向逻辑？
□ 关键假设是否做过反向测试？

偏差审计（重点检查）：
□ 确认偏误：是否刻意忽略了负面信息？
□ 质量陷阱：是否把好公司等同于好投资？
□ 故事偏误：叙事是否未经财务数字验证？
□ 过度自信：增长预测是否偏向乐观？

财务质量审计：
□ FCF与净利润是否长期匹配？
□ 有无审计异常或会计政策激进迹象？
□ 应收账款/存货增速是否异常？

审计结论：
通过 → 进入执行 Agent
通过（附条件）→ 补充特定分析后进入
不通过 → 返回表达Agent/研究Agent修正
```

**审计通过标准：**
- 核心结论有对应证据支撑
- 主要认知偏差被识别并有应对措施
- 退出条件清晰写明
- 主要风险被覆盖

---

### 阶段 6：执行落地（执行 Agent）

**输入**：审计通过的研究报告

**执行 Agent 负责：**

```
决策执行：
□ 确认投资动作（买入/持有/观察/回避）
□ 确认仓位比例
□ 设定建仓计划（一次性 vs. 分批）

跟踪机制设置：
□ 财报日提醒（下次季报/年报时间）
□ 价格预警（触发复查的价格区间）
□ 重大事件监控清单

持仓记录更新：
□ 买入价格 + 买入逻辑记录
□ 止损/退出条件记录
□ 下次复查时间
```

---

### 阶段 7：沉淀（记忆 Agent）

**输入**：完整研究过程 + 执行动作

**记忆 Agent 负责更新：**

```
必须更新：
□ 公司档案（每次研究都更新）
□ 行业地图（如有行业新发现）
□ 研究结论记录（含日期、价格、结论）

选择性更新：
□ 分析框架库（如发现新的有效框架）
□ 信息来源评级（如发现高质量新来源）
□ 失误记录本（如更新了以往的错误判断）
```

## 三、快速筛查流程（30分钟版）

当时间有限，或需要快速判断一批标的时，
使用以下简化流程：

```
第1步（5分钟）：商业模式理解
→ 这家公司怎么赚钱？护城河在哪里？

第2步（10分钟）：财务快照
→ 近3年营收增速 + 净利率 + FCF是否为正

第3步（5分钟）：估值直觉
→ 当前PE/PB处于历史什么位置？贵/合理/便宜？

第4步（5分钟）：负面筛查
→ 有无重大负面新闻/审计问题/监管风险？

第5步（5分钟）：结论
→ 值得深入研究 / 列入观察 / 跳过

输出：一句话结论（不超过50字）
```

## 四、研究效率优化原则

### 原则一：时间盒管理
每轮研究必须设定明确的时间上限：
- 初步筛查：最多2小时
- 标准研究：最多6小时
- 深度研究：最多3个工作日

超时不等于质量更好，通常意味着研究没有聚焦。

### 原则二：问题驱动而非信息驱动
研究的起点是**问题**，不是信息搜集。
在开始看数据之前，先明确：**这次研究最想回答的3个问题是什么？**

### 原则三：80%原则
一份80分的研究报告，能支持正确决策。
追求100分的研究报告，经常导致"分析瘫痪"。
**好的研究判断什么时候"够用了"。**

### 原则四：先写结论，再找证据（用于测试）
在完成数据搜集后，先写出初步结论，
然后由监督审计 Agent 检查这个结论是否有充分证据支撑。
这个逆向测试是发现确认偏误的有效方法。

### 原则五：异步消化
复杂的研究报告，不要当天读完就做决策。
让信息沉淀24小时，再做最终判断，通常质量更高。

## 五、跨公司研究的效率复用

当研究过多家公司后，记忆 Agent 沉淀的资产可以提升后续研究效率：

```
已有行业地图 → 新公司研究可跳过行业入门阶段
已有竞争格局 → 竞争分析部分可以直接更新而非重建
已有估值框架 → 同类公司估值可以基于历史框架快速推算
已有管理层档案 → 管理层评估可以基于历史记录快速更新
```

这就是研究工作室的**复利效应**：
研究越多，单次研究的边际成本越低，质量越高。

估值分析是否有依据    |         |         |
风险是否被充分识别    |         |         |

## 七、一句话总结

**股票研究工作流的核心价值，是把"看股票"这件事，
从随机、低效、依赖情绪的个人行为，
变成结构清晰、可复盘、会积累的最小专业研究系统。**

# 七核角色系统总览 v1

## Role System Overview

七核角色系统，是一人制公司智能经营系统的运行层。

它的核心逻辑是：

**每一个经营动作，都应该由一个明确的角色负责。**
**每一个角色，都应该有清晰的职责边界、输入输出与协作关系。**

---

## 一、七核角色一览

| 核 | 角色名称 | 核心职责 | 核心价值 |
|---|---|---|---|
| 1 | 判断 Agent | 判断问题是否成立、值不值得做 | 减少错误动作 |
| 2 | 编排 Agent | 把任务拆解为结构与阶段 | 让任务靠结构推进 |
| 3 | 研究 Agent | 搜集、提炼、标注信息 | 让系统更接近真实 |
| 4 | 表达 Agent | 把原材料转为可交付成果 | 把内容变为可使用 |
| 5 | 执行 Agent | 推进动作、更新状态、回传结果 | 让结果真正发生 |
| 6 | 监督审计 Agent | 检查越界、漏项、质量偏移 | 让系统值得信任 |
| 7 | 记忆 Agent | 归档过程、沉淀模板、提炼资产 | 让系统持续变强 |

---

## 二、七核的运行逻辑

七核不是七个孤立角色，而是一个有顺序、有交接的运行链路。

**基本运行顺序：**

```
判断 → 编排 → 研究 → 表达 → 执行 → 监督审计 → 记忆
```

每一核都有：
- **明确的前置输入**：从哪里接收任务
- **明确的处理职责**：自己负责什么
- **明确的输出物**：交出什么给下一核

---

## 三、每张角色卡包含什么

每张角色卡的基本结构如下：

1. **角色定义**：这个角色是什么
2. **核心职责**：它负责什么
3. **输入**：它从哪里接收输入
4. **输出**：它向哪里交出输出
5. **判断标准**：它用什么标准做决策
6. **常见误区**：这个角色最容易出的问题
7. **与主经营智能体的关系**：如何配合中枢
8. **研究场景特别说明**：在研究任务中的具体工作方式

---

## 四、七核与主理人的关系

七核系统是主理人的智能岗位系统，而不是替代主理人。

**主理人负责：**
- 确定目标与边界
- 最终优先级裁定
- 最终结论与责任

**七核负责：**
- 把任务组织起来
- 把运行链路打通
- 把结果沉淀下来

---

## 五、七核与主经营智能体的关系

```
主理人
  ↓
主经营智能体（调度、汇总、风险上浮、经营反馈）
  ↓
七核岗位系统（判断、编排、研究、表达、执行、审计、记忆）
```

主经营智能体是经营中枢，七核是运行层。

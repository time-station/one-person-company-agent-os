# 任务进入模板 v1

## Task Intake Template

---

## 使用说明

这个模板的目的是：

**让每一个进入系统的任务，都有清晰的初始描述，而不是模糊地丢给系统一句话。**

在使用七核系统之前，先用这个模板把任务写清楚。
一个填写完整的任务进入模板，是整个系统能够高质量运行的起点。

---

## 任务进入模板

```
任务名称：
[一句话，让人一看就知道这是什么任务]

任务日期：
[YYYY-MM-DD]

---

一、这个任务要解决什么问题？

[用 1-3 句话描述，这个任务的核心问题是什么。
不要写"我想做一个研究"，要写"我想搞清楚 X 是否正在发生，以及如果是，我应该如何应对"。]


二、成功的标准是什么？

[完成这个任务之后，应该有什么结果？
用具体、可判断的方式描述。
例如："形成一份 1-2 页的研究摘要，能回答 X 的核心问题"，
而不是"做一个好的研究"。]


三、时间要求

预期完成时间：[YYYY-MM-DD 或"本周内"等]
硬截止时间（如有）：[YYYY-MM-DD]


四、资源边界

可用时间：[大约多少小时]
是否有预算：[有 / 无 / 说明]
可用工具或来源：[列举主要工具或来源限制]


五、边界说明

这次任务 **不** 包括：
- [明确说明不在范围内的内容]
- [...]

这次任务的决策权限：
□ 执行前不需要主理人确认
□ 执行前需要主理人确认（请说明哪些步骤）：[___]
□ 全程需要主理人介入


六、主要风险（可选，先简单说）

[有没有什么显而易见的风险？
例如："信息可能很难搜到"，"判断可能很主观"，"这个问题本身可能没有答案"。
如果暂时不清楚，可以留空，让判断 Agent 来处理。]


七、背景信息（可选）

[有没有对这个任务有帮助的背景信息？
例如：过去的相关研究、相关来源、已知的关键事实等。]


八、主理人补充说明（可选）

[任何不属于以上类别，但主理人想说的内容。]

```

## 填写示例（研究场景）

```
任务名称：
研究 AI Agent 在一人制公司场景中的实际采用现状

任务日期：
2026-03-19

---

一、这个任务要解决什么问题？

搞清楚目前真实世界中，有哪些一人公司/超级个体在实际使用 AI Agent 进行经营，
他们在用什么工具、解决什么问题、遇到什么卡点。
这个研究用于帮助我们验证产品方向的真实需求。

二、成功的标准是什么？

形成一份研究摘要（1-2页），包含：
1. 3-5个真实用户案例或场景描述
2. 目前主要工具的使用格局
3. 最高频的卡点和未被满足的需求
4. 一个明确的"我们看到的机会"结论

三、时间要求

预期完成时间：本周内（3天）
硬截止时间：无

四、资源边界

可用时间：约 4-6 小时
是否有预算：无
可用工具或来源：公开信息、社区讨论、产品文档

五、边界说明

这次任务不包括：
- 不需要做用户访谈（本次只做桌面研究）
- 不需要覆盖企业级场景（只聚焦一人公司/小团队）

执行前需要主理人确认：
□ 研究摘要的初稿完成后，需要主理人过目

六、主要风险

- 公开信息可能不够准确，很多"真实用例"可能是宣传内容
- 这个领域变化很快，一个月前的信息可能已经过时

七、背景信息

我们之前看过 GPT 官方的 use cases 页面，以及一些社区的讨论。
重点想看的是：不是"企业如何用 AI"，而是"一个人如何靠 AI 运营一家公司"。
```

## 快速版（轻任务使用）

```
任务名称：[___]
核心问题：[这个任务要解决什么？]
成功标准：[做完了应该有什么？]
时间要求：[什么时候要？]
关键边界：[不包括什么？]
```

---

## 使用建议

1. **先填再进**：在把任务交给主经营智能体之前，先把这个模板填完
2. **不完整就停**：如果某个核心栏填不上，说明任务本身还没想清楚
3. **根据任务类型选模板**：轻任务用快速版，标准任务和高风险任务用完整版
4. **诚实填写**：不要写"看起来专业"的内容，要写真实情况

---

## 一句话总结

**任务进入模板，是让每一个任务在进入系统之前先想清楚，而不是把混乱带进系统。**

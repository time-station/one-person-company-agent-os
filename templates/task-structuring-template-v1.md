# 任务结构化模板 v1

## Task Structuring Template

---

## 使用说明

任务进入模板解决"这个任务是什么"，
任务结构化模板解决"这个任务怎么拆、怎么跑"。

**这个模板由编排 Agent 使用，也可以由主理人直接使用。**

填完这个模板，任务就有了可以被系统执行的结构。

---

## 研究任务结构化示例

```
任务名称：研究 AI Agent 在一人制公司场景中的实际采用现状
结构化日期：2026-03-19
基于任务进入模板：是

---

一、核心问题

目前真实世界中，谁在用 AI Agent 运营一人公司，他们在用什么方式、解决什么问题？

二、任务分级

☑ 标准任务

三、阶段划分

阶段 1：问题定义与范围确认
- 目标：确认研究的具体问题边界和信息类型
- 负责核：判断 Agent + 编排 Agent
- 预计时间：30 分钟
- 阶段完成标志：有清晰的子问题清单和来源搜索策略

阶段 2：桌面研究
- 目标：搜集并提炼 3-5 个真实案例，梳理工具格局和主要卡点
- 负责核：研究 Agent
- 预计时间：3-4 小时
- 阶段完成标志：有研究输入包，包含关键事实、案例和不确定性标注

阶段 3：研究摘要撰写
- 目标：形成 1-2 页结构化研究摘要
- 负责核：表达 Agent
- 预计时间：1 小时
- 阶段完成标志：初稿完成，主理人审阅

阶段 4：审阅与沉淀
- 目标：主理人确认，记忆 Agent 沉淀
- 负责核：主理人 + 记忆 Agent
- 预计时间：30 分钟
- 阶段完成标志：最终版完成，来源已存档

四、主理人仲裁点

☑ 阶段 2 完成后，主理人确认：研究发现的方向是否符合预期
☑ 阶段 3 完成后，主理人确认：初稿是否回答了核心问题

五、研究需求

研究核心问题：
谁在用 AI Agent 运营一人公司，他们在用什么方式、解决什么问题？

子问题：
1. 有哪些有公开记录的真实案例或场景？
2. 目前最常用的工具/平台是哪些？
3. 最高频的使用场景和卡点是什么？
4. 有没有什么"意外发现"（与预期不同的信息）？

需要的信息类型：
☑ 案例与故事
☑ 行业分析与观点
□ 数据与数字（如有）

研究深度：
☑ 标准研究

六、输出物规格

输出类型：☑ 研究摘要
输出长度：约 1-2 页
输出格式：Markdown
输出语言：中文

最终使用方式：☑ 主理人内部参考
```

## 一句话总结

**任务结构化模板，是让每一个任务不靠临场决定"下一步怎么做"，而是在开始之前就把路走清楚。**

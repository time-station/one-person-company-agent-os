# Execution Log Template v1

## 执行留痕模板

这个模板的目标，不是让系统写很多过程报告，  
而是留下最必要的执行痕迹，避免任务推进后只剩结果，没有过程可回看。

执行留痕的意义在于：

- 让关键动作可追溯
- 让断链位置可识别
- 让异常情况可回收
- 让后续复盘有依据

它不是为了增加负担，  
而是为了让任务推进变得更可信。

---

## 一、任务基本信息

### 任务名称
一句话说明任务是什么。

### 任务模式
- 轻任务
- 标准任务
- 高风险任务

### 主理人
谁拥有最终判断与责任。

### 当前执行节点
当前推进到哪个阶段。

---

## 二、执行记录

### 记录时间
记录这次动作发生的时间。

### 执行动作
一句话写清楚这次做了什么。

### 执行主体
是谁完成了这一步：
- 主经营智能体
- 判断 Agent
- 编排 Agent
- 研究 Agent
- 表达 Agent
- 执行 Agent
- 审计 Agent
- 记忆 Agent
- 主理人

### 输入是什么
这一步的输入依据是什么。

### 输出是什么
这一步产生了什么结果。

---

## 三、状态判断

### 当前状态
- 正常推进
- 需要补信息
- 出现阻塞
- 需要主理人拍板
- 已完成当前节点

### 是否发生偏移
- 无
- 有轻微偏移
- 有明显偏移

如果有偏移，一句话说明偏在哪里。

---

## 四、异常记录

### 是否出现异常
- 否
- 是

### 异常类型
例如：
- 目标不清
- 输入不足
- 交接断链
- 外部动作受阻
- 输出质量不达标
- 越界风险上升

### 处理方式
一句话说明当前如何处理。

### 是否需要上浮
- 不需要
- 需要主理人判断
- 需要提高审计强度

---

## 五、下一步

### 下一步动作
下一步最应该发生什么。

### 下一步责任主体
下一步由谁接手。

### 是否需要补充审计
- 不需要
- 需要轻量审计
- 需要高强度审计

---

## 六、最小使用方式

如果不想写完整模板，  
至少也要保留这六项：

1. 记录时间
2. 执行动作
3. 执行主体
4. 当前状态
5. 是否异常
6. 下一步责任主体

---

## 一句话总结

**执行留痕模板，是为了让关键动作、状态、异常与交接在任务推进中留下最小可追溯记录。**

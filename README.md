# 让一人制公司真正拥有一套智能经营系统

一个开源项目，探索一人制公司如何通过主经营智能体（Chief Operating Agent）与七核岗位系统，构建最小智能经营系统。

---

## 这不是一个普通的 AI 工具仓库

我们关心的，不只是“一个人如何使用更多 AI 工具”，  
而是更深一层的问题：

**一个现实世界中的一人制主理人，如何通过智能世界中的 Agent 能力，形成真正可经营、可协作、可执行、可审计、可沉淀的最小经营系统。**

也就是说，我们想推动的不是“多几个助手”，  
而是：

- 更清楚的主权结构
- 更真实的经营代理
- 更可运行的岗位协作
- 更有复利的能力沉淀

---

## 这个项目在探索什么

这个项目围绕一条主线展开：

**主理人  
→ 主经营智能体（Chief Operating Agent）  
→ 七核岗位系统**

一句话理解：

**主理人掌握方向，主经营智能体组织经营，七核岗位系统负责运行。**

这套结构希望解决一人制公司最真实的问题：

- 事情太多，但组织半径有限
- 能力很多，但没有被组织起来
- 项目在推进，但没有最小经营秩序
- 做了很多事，但没有留下可复用资产

---

## 为什么现在值得做这件事

因为越来越多个人与小团队，已经站在一个新拐点上：

一方面，  
一人制公司、超级个体、微型创业与轻组织形态正在持续出现。

另一方面，  
Agent 正在从工具走向岗位、从岗位走向协作、从协作走向经营代理。

这两股力量一旦结合，
带来的不只是效率提升，  
而是新的生产方式、新的经营方式，甚至新的创业结构。

这个项目，就是想为这种变化，先立一个开源起点。

---

## 仓库里现在有什么

当前仓库主要由四层构成：

### `manifesto/`
项目的愿景、使命、核心原则与总纲。  
回答：**我们为什么做这件事。**

### `foundations/`
一人制公司智能经营系统的方法骨架。  
回答：**这套系统是什么结构。**

### `protocols/`
最小监督审计、任务审计、执行留痕、任务复盘等治理结构。  
回答：**这套系统如何保持秩序与复利。**

### `community/`
社区指南、共建路径、讨论主题。  
回答：**后来者如何进入、如何共建。**

### `use-cases/`
内容工作室、研究工作室、项目推进系统等样板。  
回答：**这套系统在真实场景中怎么跑。**

## 我们相信什么

我们相信：

- 一人制公司不是小公司缩小版，而是主权清晰的微型智能组织
- Agent 的价值不在于更多，而在于更有组织
- 真正的竞争力不是单点提效，而是组织智能生产力
- 系统变强的关键，不只是完成任务，而是沉淀资产
- 不迷信智能，优先组织智能
- 不只追求效率，更要形成经营能力

# 七核交接清单 v1

## Seven-Core Handoff Checklist

---

## 一、交接清单的存在意义

七核协作工作流定义了"整体怎么跑"，
七核交接清单定义了"每一个交接点具体交什么"。

**交接清单的目标：让每一次交接都清晰、完整、可验证。**

---

## 二、通用交接原则

在每一个交接点，交出方需要确认：

```
□ 我的输出是否完整？（没有遗漏必要内容）
□ 我的输出是否清晰？（接收方能直接使用）
□ 我的输出是否已经标注不确定性？
□ 我是否说明了需要接收方特别注意的地方？
□ 我是否已经记录了这次交接？（留痕）
```

---

## 三、各核交接清单

### 3.1 主理人 → 主经营智能体

**主理人传递时，需要说清楚：**

```
□ 这次任务的核心意图是什么？（一句话）
□ 成功的标准是什么？（完成后应该有什么结果）
□ 时间边界是什么？（什么时候需要）
□ 资源边界是什么？（有什么限制）
□ 哪些是不能碰的边界？（禁止的范围）
□ 需要主理人在哪些节点介入？（提前说明仲裁偏好）
```

---

### 3.2 主经营智能体 → 判断 Agent

**主经营智能体传递时，需要说清楚：**

```
□ 任务描述（已从主理人意图翻译成清晰任务）
□ 任务背景（为什么做这件事）
□ 主理人的核心关切
□ 时间和资源约束
□ 需要判断的核心问题是什么
```

---

### 3.3 判断 Agent → 编排 Agent

**判断 Agent 交出时，需要包含：**

```
□ 判断结论：做 / 不做 / 暂缓（并说明原因）
□ 任务分级：轻任务 / 标准任务 / 高风险任务
□ 任务核心问题（最重要的一个问题是什么）
□ 任务边界（不在范围内的内容）
□ 主要风险清单（3条以内，每条有应对建议）
□ 特别注意事项（编排时需要考虑的特殊因素）
```

---

### 3.4 编排 Agent → 研究 Agent

**编排 Agent 传递研究任务包时，需要包含：**

```
□ 研究核心问题（一句话）
□ 子问题清单（2-5个）
□ 需要的信息类型（定量数据？行业分析？案例？观点？）
□ 来源偏好（有没有特别指定的来源类型）
□ 研究分级（轻 / 标准 / 深，决定时间投入）
□ 研究输出格式要求（交给表达 Agent 的格式）
□ 截止时间
```

---

### 3.5 研究 Agent → 表达 Agent

**研究 Agent 交出研究输入包时，需要包含：**

```
□ 核心问题已回答（或说明无法回答的原因）
□ 关键事实清单（每条有来源和确定性标注）
□ 关键变量清单
□ 主要矛盾或争议点
□ 不确定性清单（哪些信息不确定，哪些需要进一步求证）
□ 高价值来源列表（带简短说明）
□ 研究发现中的"意外"（与预期不符的发现）
```

---

### 3.6 表达 Agent → 执行 Agent

**表达 Agent 交出成果时，需要包含：**

```
□ 最终输出物（报告 / 摘要 / 备忘录等）
□ 核心结论（1-3句话，明确）
□ 不确定性说明（哪些判断有较大不确定性）
□ 建议的后续动作（具体的行动建议）
□ 输出需要进一步处理的说明（如需要格式化、发送等）
□ 特别注意事项（执行时需要留意的地方）
```

---

### 3.7 执行 Agent → 主经营智能体

**执行 Agent 回传时，需要包含：**

```
□ 执行结果（做了什么，结果是什么）
□ 当前状态（完成 / 进行中 / 遇到阻碍）
□ 如有阻碍：具体说明阻碍内容和需要什么支持
□ 确认留痕（执行记录在哪里）
□ 下一步建议（接下来应该做什么）
```

---

### 3.8 监督审计 Agent → 主经营智能体

**监督审计 Agent 输出审计报告时，需要包含：**

```
□ 总体质量评级（优 / 良 / 待改进 / 有重大问题）
□ 发现的主要问题清单（每条有严重程度和建议）
□ 留痕完整性确认
□ 越界行为记录（如有）
□ 结构性改进建议（对系统层面的建议）
□ 是否建议继续 / 暂停 / 重做
```

---

### 3.9 记忆 Agent → 系统资产库

**记忆 Agent 完成沉淀时，需要确认：**

```
□ 任务归档包已完成（含过程记录）
□ 新模板已添加到模板库
□ 旧模板是否需要更新（已检查并处理）
□ 高价值来源已添加到来源库
□ 规则更新建议已记录（待主经营智能体审阅）
□ 后续跟踪事项已记录
□ 经验摘要已完成
```

---

## 四、研究场景的特殊交接说明

研究任务的交接，特别需要注意两个地方：

### 研究 Agent → 表达 Agent 的交接

这个交接最容易出问题，因为：
- 研究 Agent 的输出是"材料"
- 表达 Agent 需要的是"可以讲一个故事的材料"

**交接时需要特别说明：**
- 哪些发现最出乎意料（往往是最有价值的部分）
- 哪些矛盾最需要在报告中正视
- 哪些不确定性最影响最终结论

### 表达 Agent → 执行 Agent 的交接

研究成果需要进入现实，这个交接决定了研究是否真正发挥作用。

**交接时需要特别说明：**
- 这份研究成果希望产生什么现实影响
- 发给谁，以什么方式
- 是否需要等待主理人确认再发出

## 五、快速检查版（任务结束后使用）

```
任务名称：_______________
完成日期：_______________

七核运行情况检查：
□ 判断 Agent 完成，有输出记录
□ 编排 Agent 完成，有结构文档
□ 研究 Agent 完成，有研究输入包
□ 表达 Agent 完成，有最终输出物
□ 执行 Agent 完成，有执行结果报告
□ 监督审计 Agent 完成，有审计报告
□ 记忆 Agent 完成，有记忆包

仲裁点执行情况：
□ 所有设定的仲裁点都已执行
□ 主理人在关键节点有确认记录

交接质量评估：
□ 每个交接都有明确的输入输出
□ 没有遗漏必要内容的交接
□ 不确定性在交接过程中被传递
```

---

## 六、一句话总结

**七核交接清单，是让七核协作不靠默契、不靠临场判断，而是靠清晰的输入输出结构稳定运行的核心工具。**

# Seven-Core Collaboration Workflow v1

## 七核协作工作流

七核模型如果只停留在定义层，
它仍然只是一个结构概念。

只有当七核真正形成协作链路，
这套系统才开始从“骨架”进入“运行”。

这份工作流的目的，就是回答一个最实际的问题：

**主经营智能体如何组织七核岗位系统，把一个真实任务跑成最小闭环。**

---

## 一、七核不是并列摆放，而是协作链路

七核包括：

1. 判断 Agent  
2. 编排 Agent  
3. 研究 Agent  
4. 表达 Agent  
5. 执行 Agent  
6. 监督审计 Agent  
7. 记忆 Agent  

它们的价值不在于“七个名字”，  
而在于它们能够在任务推进过程中形成有顺序、有交接、有反馈的协作链路。

最小链路是：

**判断 → 编排 → 研究 → 表达 → 执行 → 监督审计 → 记忆**

这不是机械流水线，  
而是一个可以根据任务模式变化强度的最小运行结构。

---

## 二、工作流的总入口

所有任务都应该先经过：

### 主理人
提出目标、问题、任务或方向。

然后由：

### 主经营智能体
判断这件事属于：
- 轻任务
- 标准任务
- 高风险任务

并决定七核要以什么强度介入。

所以工作流真正的总入口不是某一核，  
而是：

**主理人 → 主经营智能体 → 七核协作链路**

---

## 三、标准任务的默认工作流

### 第一步：判断介入
判断 Agent 先回答：

- 这件事值不值得做
- 现在该不该做
- 风险大不大
- 是否需要主理人拍板
- 这件事更适合轻跑还是完整推进

如果判断不成立，任务直接停止或延后。  
如果成立，再进入下一步。

---

### 第二步：编排介入
编排 Agent 负责：

- 把任务拆成阶段
- 明确节点
- 明确交接关系
- 明确哪些地方需要研究
- 明确哪些地方会进入表达、执行与审计

这一步的作用是：

**让任务从“一个想法”变成“一个可运行结构”。**

---

### 第三步：研究介入
研究 Agent 负责补齐：

- 必要信息
- 关键事实
- 证据来源
- 案例对比
- 不确定性说明

研究不是越多越好，  
而是要为后续表达、执行与判断服务。

---

### 第四步：表达介入
表达 Agent 负责把前面的结构与研究转成：

- 文稿
- 摘要
- 方案
- 对外说明
- 阶段成果

这一层的任务是：
**把系统中的中间材料，整理成真正可交付的结果。**

---

### 第五步：执行介入
执行 Agent 负责推动真实动作发生，例如：

- 上传文档
- 建仓库
- 更新页面
- 发布内容
- 推进下一节点
- 形成外部动作

执行层让任务真正从系统内走向现实世界。

---

### 第六步：监督审计介入
监督审计 Agent 负责检查：

- 是否越界
- 是否断链
- 是否偏离目标
- 是否遗漏关键节点
- 结果是否达标
- 是否需要主理人重新判断

它的任务不是“事后找错”，  
而是：

**在系统推进中尽早发现高代价问题。**

---

### 第七步：记忆介入
记忆 Agent 负责沉淀：

- 模板
- 规则
- 工作流经验
- 高价值来源
- 失败经验
- 样板
- 下一轮可复用资产

这一层的作用是：

**让系统每跑一轮，都不是白跑。**

---

## 四、不同任务模式下的变化

### 1）轻任务
轻任务可以压缩流程，例如：

**判断 → 表达 / 执行 → 记忆**

不必七核全开。  
目标是低摩擦完成。

---

### 2）标准任务
标准任务使用完整最小链路：

**判断 → 编排 → 研究 → 表达 → 执行 → 监督审计 → 记忆**

这是默认模式。

---

### 3）高风险任务
高风险任务要强化三层：

- 判断
- 监督审计
- 主理人拍板

也就是说，它不是简单加快推进，
而是要提高治理强度。

---

## 五、主经营智能体在工作流中的位置

主经营智能体不是某一核，  
也不直接替代七核。

它负责：

- 决定任务怎么进入系统
- 决定哪些核先介入
- 决定什么时候需要主理人回到回路
- 压缩状态给主理人
- 让整个工作流有节奏感

所以它更像：

**七核协作链路的经营组织者。**

---

## 六、最容易出问题的地方

七核协作中，最常见的问题有：

### 1）没有判断前置
导致任务一开始就跑偏。

### 2）没有结构化编排
导致研究、表达、执行互相混乱。

### 3）研究过载
看了很多，反而推进更慢。

### 4）表达与执行脱节
文稿写完了，但真实动作没有发生。

### 5）没有审计
导致越界、断链、失真没人发现。

### 6）没有记忆
导致每轮都像重来。

---

## 七、最小使用方式

如果今天只想跑通一轮最小协作链路，可以这样：

1. 主理人提出任务  
2. 主经营智能体判断任务模式  
3. 判断 Agent 先判是否值得做  
4. 编排 Agent 拆出最小结构  
5. 研究 / 表达 / 执行按需要介入  
6. 监督审计做轻量检查  
7. 记忆 Agent 回收资产  

只要这条链路开始稳定运行，
七核就不再只是理论结构，而开始变成操作系统的一部分。

---

## 一句话总结

**七核协作工作流，是主经营智能体组织判断、编排、研究、表达、执行、监督审计与记忆七个岗位，推动任务形成最小闭环的运行链路。**

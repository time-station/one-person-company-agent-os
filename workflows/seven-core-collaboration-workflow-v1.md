# 七核协作工作流 v1

## Seven-Core Collaboration Workflow

---

## 一、工作流的存在意义

七核角色系统定义了"谁负责什么"，
而七核协作工作流定义了"它们如何一起运转"。

**没有协作工作流的七核，只是七个孤立角色。
有了协作工作流的七核，才是一个能跑的系统。**

---

## 二、基本运行结构

七核协作的基本运行结构是一个闭环：

```
主理人 → 主经营智能体
            ↓
         判断 Agent
            ↓
         编排 Agent
            ↓
      研究 Agent ←→ 表达 Agent
            ↓
         执行 Agent
            ↓
      监督审计 Agent（全程）
            ↓
         记忆 Agent（结束时）
            ↓
         主经营智能体（汇总）
            ↓
          主理人（仲裁点）
```

---

## 三、完整协作流程

### 第 0 步：意图输入

**负责方：主理人**

主理人提出一个意图、问题或目标，输入给主经营智能体。

输入形式可以是：
- 一个问题："这个行业现在是什么状态？"
- 一个目标："我想在两周内写出一份竞品分析报告。"
- 一个任务："整理上周会议的所有结论和后续动作。"

---

### 第 1 步：任务翻译与初始化

**负责方：主经营智能体**

主经营智能体接收主理人意图，完成：
1. 把模糊意图翻译成清晰的任务描述
2. 评估任务大致规模
3. 决定是否进入七核系统，还是直接简单处理
4. 传递给判断 Agent

**输出：** 初始任务描述包

---

### 第 2 步：判断

**负责方：判断 Agent**

接收初始任务描述包，完成：
1. 判断任务是否值得做
2. 对任务进行分级（轻 / 标准 / 深）
3. 识别主要风险
4. 设定研究边界
5. 输出判断结论

**判断结论的三种情形：**
- **进入系统**：任务成立，进入编排
- **暂缓**：需要等待某个前置条件
- **退回**：任务本身有问题，需要主理人重新定义

**输出：** 判断结论包（含任务分级、风险提示、边界说明）

---

### 第 3 步：编排

**负责方：编排 Agent**

接收判断结论包，完成：
1. 把任务拆解为阶段与节点
2. 设计各核的协作顺序
3. 设定主理人仲裁点
4. 分发任务包给各核

**输出：** 任务结构图 + 各核任务包

---

### 第 4 步：研究

**负责方：研究 Agent**

接收研究任务包，完成：
1. 搜集相关资料
2. 提炼关键事实与变量
3. 标注来源质量与不确定性
4. 形成研究输入包

**研究深度由判断 Agent 的分级决定：**
- 轻研究：快速扫描，核心事实提炼
- 标准研究：系统搜集，多来源对比
- 深研究：全面搜集，矛盾识别，框架建立

**输出：** 研究输入包（关键事实 + 变量 + 不确定性）

---

### 第 5 步：表达

**负责方：表达 Agent**

接收研究输入包，完成：
1. 整合研究材料
2. 形成结构化输出
3. 压缩为可判断的核心结论
4. 保留不确定性标注
5. 给出下一步建议

**输出：** 结构化成果（报告 / 摘要 / 备忘录等）

---

### 第 6 步：仲裁点

**负责方：主理人（通过主经营智能体）**

在编排设定的仲裁点上，主理人或主经营智能体审阅当前进展：
- 确认继续
- 调整方向
- 暂停并重新判断

**这是系统保持主权清晰的关键机制。**

---

### 第 7 步：执行

**负责方：执行 Agent**

接收表达成果和执行任务包，完成：
1. 把结论推进到现实动作
2. 调用必要工具
3. 更新状态
4. 回传执行结果

**执行前必须确认：**
- 执行范围是否在授权内
- 是否需要主理人确认

**输出：** 执行结果报告 + 状态更新

---

### 第 8 步：监督审计

**负责方：监督审计 Agent（全程运行）**

监督审计 Agent 在整个运行过程中持续工作，特别关注：
1. 各核输出的质量
2. 责任链的清晰性
3. 越界风险
4. 留痕完整性

在任务结束后，输出完整审计报告。

**输出：** 审计报告 + 治理改进建议

---

### 第 9 步：记忆与沉淀

**负责方：记忆 Agent**

在整个任务完成后运行，完成：
1. 任务过程归档
2. 可复用模板提炼
3. 高价值来源更新
4. 规则更新建议
5. 下次跟踪事项记录

**输出：** 记忆包（归档 + 模板 + 经验摘要）

---

### 第 10 步：经营反馈

**负责方：主经营智能体**

汇总整个任务的运行结果，向主理人输出：
1. 任务完成情况
2. 质量评估
3. 时间与资源消耗
4. 下一步建议

**输出：** 经营反馈摘要

## 四、三种任务模式的工作流差异

### 轻任务模式

适合：明确、低风险、30分钟内可完成的任务

```
意图输入 → 判断（简化）→ 直接执行 → 轻量记忆
```

省略：完整编排、深度研究、完整审计

---

### 标准任务模式

适合：需要研究和结构化的常规任务

```
意图输入 → 判断 → 编排 → 研究 → 表达 → 执行 → 审计 → 记忆
```

完整运行七核，但节奏适中。

---

### 高风险任务模式

适合：涉及高成本、不可逆、对外影响的任务

```
意图输入 → 判断（强化）→ 编排（含多个仲裁点）→ 研究（深度）→ 表达 → 仲裁 → 执行 → 审计（全程）→ 记忆
```

增加：多个仲裁点、实时审计、主理人多次介入

---

## 五、协作中的常见问题

### 问题一：各核之间的交接断层
**症状：** 研究 Agent 做完了，但表达 Agent 不知道接什么
**解决：** 编排 Agent 必须明确定义每个交接点的输入输出格式

### 问题二：没有仲裁点，任务跑偏
**症状：** 执行到一半，主理人发现方向不对
**解决：** 编排 Agent 在关键节点后设置仲裁点，不要等到最后

### 问题三：记忆 Agent 被跳过
**症状：** 任务完成后，没有沉淀，下次类似任务又从零开始
**解决：** 记忆 Agent 是任务完成的必要步骤，不是可选项

---

## 六、与主理人主权的关系

整个工作流的设计围绕一个核心原则：

**主理人始终保持对系统的最终主权。**

主理人的主权体现在：
- 在仲裁点介入
- 随时调整方向或暂停任务
- 对最终结论负责
- 不需要管理系统内部细节，但可以在任意节点介入

---

## 七、一句话总结

**七核协作工作流，是让一人制公司的每一个任务，都有结构地进入、有逻辑地运行、有闭环地结束，并在每次结束后变得更强。**

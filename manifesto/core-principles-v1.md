# Core Principles v1

这套系统的最小骨架，建立在四条核心原则之上。

---

## 1. 主权不可分散

Agent 可以分工，但不能夺走主权。

目标、边界、优先级、关键仲裁与最终责任，
必须回到主理人。

我们不把系统设计成“替代主理人”，
而是设计成“放大主理人”。

---

## 2. Agent 必须岗位化

我们关心的不是堆更多助手，
而是形成真正可运行的岗位系统。

每个 Agent 都应当有明确职责、输入、输出、边界与交接关系。

这意味着：

- 判断要前置
- 编排要清楚
- 执行要留痕
- 审计要存在
- 记忆要沉淀

不是角色越多越好，
而是岗位关系越清楚越好。

---

## 3. 运行必须闭环化

一个真正可经营的系统，
不能只有“生成”和“执行”，
还必须形成最小闭环。

我们强调的闭环是：

**判断 → 编排 → 研究 → 表达 → 执行 → 监督审计 → 记忆**

如果没有审计，系统会失真。  
如果没有记忆，系统不会复利。  
如果没有判断，系统会忙错方向。

---

## 4. 资产必须沉淀化

每次任务不应只追求一次性完成，
还应尽可能沉淀为未来可复用的能力资产。

这些资产包括：

- 模板
- 工作流
- 协议
- 角色卡
- 高价值来源
- 复盘经验
- 经营反馈

系统真正变强，不只是因为做了更多事，
而是因为留下了更多可复用结构。

---

## 一句话总结

**主权归主理人，能力组织成岗位，运行形成闭环，结果沉淀为资产。**
